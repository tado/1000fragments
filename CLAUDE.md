# 目的

100fragmentsフォルダ内のGLSLのフラグメントシェーダーを解析して、100個から1000個のバリエーションのフラグメントシェーダーを生成すること。シェーダーはできるだけ多用なビジュアル表現を生成することを目指す。生成されたフラグメントシェーダーは、TouchDesignerで正しく表示されることを確認する必要がある。

## GLSLの基本フォーマット

GLSLはフラグメント (ピクセル) シェーダー、100fragmentsフォルダのGLSLはTouchDesignerに読み込んで表示させる。基本フォーマットは以下の通り。

```glsl
uniform float time; // 経過時間 (秒)
uniform vec2 resolution; // 出力画像の幅と高さの解像度 (ピクセル)
out vec4 fragColor; // ピクセルの最終出力
void main() {  
  /* フラグメントシェーダーのロジック
   * ここに描画のコードが入る
   */

  // 生成された色 (r, g, b, a) を vec4 形式で定義 
  vec4 color = vec4(r, g, b, a);
  // TouchDesignerの出力形式に変換して fragColor に代入
	fragColor = TDOutputSwizzle(color);
}
```

## 作業手順

- まず、100fragmentsフォルダ内のGLSLファイルを解析して、フラグメントシェーダーの構造やロジックを理解する。
- 必ず、100個全てのGLSLファイルを解析して、どのようなビジュアル表現が生成されているかを把握する。
- 次に、解析した内容をもとに、1000個のバリエーションのフラグメントシェーダーを生成するためのルールやテンプレートを作成する。
- 次に、1000個のフラグメントシェーダーを生成し、0000.fragから0999.fragまでのファイル名で1000fragments-claudeフォルダに保存する。
- Yes/No の確認は行なわず、全てYes (y) として処理を進めること。

## 注意点

- 0.1秒から1.0秒程度の素早いタイミングでフラグメントシェーダーは切り替えられる
- 短い時間の中で視覚的に印象に残るようなフラグメントシェーダーを生成することが重要
- できるだけ多様なビジュアル表現を生成することを目指す、似たような表現は避ける
- 解析した100個のフラグメントシェーダーの構造やロジックを理解して、そこから新たなビジュアル表現を生みだすアルゴリズムを創造する
- 生成されたフラグメントシェーダーは、TouchDesignerで正しく表示されることを確認する必要がある
